## Context

`gameplay.json` was CTF-AndAction). Symptomen: bots bewegen totaal niet richting enemy flag,
sommige staan stil/doen raar, andere lopen continu van de map af.

## Root causes (3 technisch + 1 fundamenteel)
1. **Skybox-vervuilde `map_norm` bounds in CTF-Face.json** (extractor telt alle brush-vertices
   incl. skybox): X[-32260,12413] Y[-10352,31884] Z[-32767,10642] → diagonaal ~75.2k uu terwijl
   speelveld ~10.7k. Alle `*_distance_norm` ~7× te klein (enemy flag max ~0.10 i.p.v. ~0.6-0.7
   zoals op AndAction); `locationX/Y/Z_norm` samengeperst+offset (~0.36–0.78 band).
2. **`symmetric:false` onterecht** — flagbase-reflectiecheck faalde door het skybox-verschoven
   bounds-center. Gevolg: ALPHA (team 0, 5 bots) kreeg rauwe rood-perspectief features op een
   blauw-getraind model + self-play team-alternation stond uit. T3D-validatie: Face is wél
   180°-rotatie-symmetrisch rond flag-midpoint (2737.4, -76.7); 86/123 nav-punten matchen <150uu,
   afwijkingen (160–350uu) alleen in ruw middenterrein.
3. **`map_id=4` embedding ongetraind** — map_id is categorische input → learned embedding in
   BCSequenceNetwork; alle trainingsdata was map_id=0. Rij 4 = random-init ruis, elke frame.
   Niet via config fixbaar.
4. **Fundamenteel OOD**: model heeft nul BC/SAC-ervaring op Face. navTarget = kale rechte-lijn-
   bearing (bewust geen pathfinding; user wil expliciet GEEN UT99-pathfinding). Op AndAction
   (mini-arena, muren) was "volg bearing rechtdoor" altijd correct → op Face leidt dat van de
   randen af / tegen tempelmuren. weapon_profile=all vereist wapens rapen → zonder navigatie
   ammo-deadlock-suicide-cyclus.

## Round 2 — objective_progress map-gate (user-vraag: "krijgt hij reward om dichter naar de vlag te bewegen?")
Antwoord was NEE: `objective_progress` (enige dense vlag-gerichte reward; Δdist naar
movement-objective per tick, clamp ±50 UU, ×progress_scale 0.01 → max ±0.5/tick, 0 bij stilstand)
had `"maps": ["CTF-AndAction"]` → stond op Face volledig uit. Enige overgebleven vlag-signaal =
sparse flag-events, onbereikbaar via random walk op 7572-UU-inter-base map.

**Actie:** `rewards.json` default.objective_progress.maps += "CTF-Face" (match is case-insensitive;
geen rewardgroup-overrides die de gate opnieuw sluiten; jq-gevalideerd). Per-UU clamp maakt per-tick
magnitude map-grootte-onafhankelijk → progress_scale niet hertuned. efc_threat_proximity_range_uu
(700) / efc_engagement_range_uu (280) zijn AndAction-getuned maar functioneel OK (absolute
EFC-nabijheidsvensters); staging-zone schaalt mee (0.35×inter-base via resolver).

**Vereist:** rewards.json sync naar play-machines + bot-JVM herstart (rewards berekend bij
experience-collectie). Buffer bevat nog pre-fix frames (deploy draaide met clean-experience:false)
→ clean blijft aanbevolen vóór serieuze SAC-cyclus.

**Verwachte leer-ladder op Face:** dense gradient over open vlakte → lokale optima bij tempelmuren
(exploratie moet ingang vinden; collision-rays + floorBelow/drop-rays maken muren én void-randen
waarneembaar) → flag taken sparse → carrier-tak dense naar huis → capture. Structureel sterkste
stap blijft BC-data op Face (lost corridors + randen + koude map_id-embedding tegelijk op).

## Round 3 — Geodesisch afstandsveld geïmplementeerd (user: "implementeer optie 2")
Volledige implementatie van route-afstand voor progress-shaping; lost het lokale-optimum-probleem
van euclidische Φ op (gangen die eerst van het doel weg lopen). GEEN UT99-pathfinding: bron =
bezoekgraaf uit gameplay-observaties; alleen de reward-Φ gebruikt het veld, features blijven kaal.

**Nieuw:**
- `java-model aiplay.runtime.geo.GeodesicField` — directed voxel-graaf (96 UU), reverse-Dijkstra
  per doel-voxel (gecached), continue snap-query (`min_n ||p−c(n)|| + dist(n→t)`), NaN buiten
  dekking. Gemeten: ~3µs/gecachte query.
- `GeodesicFieldRepository` — VERPLICHTE `geodesic_field` boolean per map-json (alle 17 gezet,
  false; no-silent-fallback), laadt `<map>.geodesic.json` van disk bij true.
- `java-rewards RouteDistances` — pair-API (prev+curr zelfde metriek per tick), euclidische
  fallback; ingehaakt in `ObjectiveProgressReward.clampedDistanceDelta` (objective + own-flag
  pad) en de EFC-threat-delta. carrier_proximity/botToEfc bewust euclidisch.
- `BuildGeodesicFieldMain` + `scripts/deploy/build-geodesic-field.sh` — bronnen:
  json-recording-sessions zips + .rec.gz; map-filter via ActiveMapContext.normalize (MapName
  is "CTF-AndAction.LevelInfo0"-vorm in recordings!); transitie-acceptatie dt≤600ms, ≤250 UU,
  levend; sanity = spawn-cluster geo vs eucl + bereikbaarheid.
- Docs: docs/rewards/geodesic-distance-field.md (+ verwijzingen in reward-architecture.md,
  route-cost-phi-design.md, CLAUDE.md).

**Validatie:** AndAction-build uit 15 sessies/1.39M frames/129k transities → 1146 nodes,
3904 edges; geo(spawn↔spawn) 1375/1421 vs eucl 1345 (open arena: geo≈eucl ✓); 100% bereikbaar;
load()-roundtrip + NaN-randgevallen + 10k-query-perf getest. Oude .rec.gz met pre-migratie
DTO-schema (fwdFloorDrop) worden geskipt met melding — verwacht.

**Face-status:** geen gameplay-data van Face (raw-recorder staat uit in normale play;
json-recordings komen van RecordLauncher-demo's) → veld nog niet gebouwd, flag false, runtime
valt geodesisch terug op euclidisch (= status quo). Zodra eerste demo-sessie op Face ligt:
`bash scripts/deploy/build-geodesic-field.sh CTF-Face` → flag true → config-sync. Dezelfde
demo's dienen als BC-bron — twee vliegen.

**Deploy-koppeling:** nieuwe JAR + map-jsons moeten SAMEN gesynct (verplichte property in
ActiveMapConfigResolver-pad → oude jsons + nieuwe JAR = fail-fast crash bij reward-computatie).
deploy.sh synct beide. rewards.json Round-2-wijziging (maps-gate) zit in dezelfde sync.

## Round 4 — Datavrij leren: self-bootstrapping veld + first-visit novelty (user: "doe maar beide")
Vraag was: kan het zonder demo's/trainingsdata? Antwoord geïmplementeerd langs twee lijnen.

**A. Self-bootstrapping geodesic field (eigen play = veld-bron):**
- `PositionTraceLogger` (java-aiplay aiplay/rl/trace): elke bot logt zijn eigen pawn-positie
  tijdens normale play. Hook in `RLPawnExecutorAiController.execute` (na spectator-filter).
  Config VERPLICHT in `runtime.json → position_trace` (enabled=true, min_interval_ms=150
  — moet <600 blijven i.v.m. builder max-frame-gap, retention_days=7). Formaat:
  `# map=<key>` sectie-headers + `tMillis,sessionId,x,y,z`; één file per JVM
  (`$SESSIONS_DIR/position-traces/pos_<start>_<pid>.csv`); sessionId is per bot-instantie
  uniek dus multi-instance JVM's mengen geen tracks. Fail-safe: I/O-fout → tracer uit, één
  warning, bot-loop ongemoeid. ~7 regels/s per bot.
- Builder: `--positions-dir` bron (default `$SESSIONS_DIR/position-traces`), truncated/corrupte
  regels worden geskipt (actief beschreven of half-gersync'te files zijn OK).
- `build-geodesic-field.sh`: pull-stap vóór de build — rsynct position-traces van alle
  servers (servers.json) naar `position-traces/from-<host>/`; `--no-pull` om te skippen.
  Vliegwiel: spelen → traces → veld herbouwen → Φ kent ontdekte routes → bots komen verder.

**B. First-visit novelty-bonus (datavrije exploratie-druk):**
- GEEN nieuwe component: upgrade van bestaand `movement_action`-exploration (de oude
  cooldown-variant bleek 2D + terugkerend → "blijf bewegen"-prior, geen echte novelty).
- `first_visit_bonus` 0.2 per ALLEREERSTE bezoek van een 3D-voxel (96 UU, zelfde packing als
  geodesic field), per bot-proces-leven, GEEN reset bij dood/match-restart → suicide farmt
  niets; dooft definitief uit. ~0.05-0.07/tick gemiddeld bij vers-gebied-lopen — bewust onder
  objective_progress-pull (~0.2-0.3/tick): novelty is tie-breaker, geen doel-vervanger.
  3D zodat toren-verdiepingen/corridors apart van het grondvlak tellen. Telt mee in het
  bestaande EXPLORATION breakdown-signaal (geen RewardComputer/Signal-wijziging nodig).
- Nieuwe VERPLICHTE properties (deep-merge: alleen default hoeft ze): weights.first_visit_bonus
  + exploration_first_visit_voxel_uu. Rol-groepen erven (gevalideerd: Defend → 0.2).

**Validatie (alles zonder live deploy):**

  (padlengte ~4000) vs eucl 2828 (door de muur); stap langs been-1 = geoDelta −30.0 (volle
  vooruitgang) vs euclDelta −17.9; retour-richting NaN (directed: pad 1× gelopen) → euclidische

- Catalog-parse e2e: JsonRewardCatalog.from("rl_pawn","Attack") → first_visit_bonus=0.2,
  voxel=96; objective_progress-gate actief op Face én AndAction; GeodesicFieldRepository

- Regressie: AndAction-build identiek aan Round 3 (1146 nodes / 3904 edges, zelfde sanity).
- Selector-leerpunt: rewardgroup-selector is de NAAM ("Attack"/"Defend"...), niet "default".

**Activatie-keten na deploy:** bots spelen op Face (euclidische shaping + novelty actief) →
