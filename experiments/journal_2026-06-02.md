## Round 1 (03:00) — Dense combo-discovery shaping (SHOCK_COMBO_AIM) + sparse bump

**Hypothese:** een dense per-tick reward voor "beam (fireActive) gericht op je eigen live ShockProj,
terwijl die bal vlakbij een enemy is" geeft het beleid de ontbrekende gradient om de combo-timing
te ontdekken — zonder op toevallige successen te wachten. Generaliseert: triggert ALLEEN bij een
eigen `Botpack.ShockProj` (0 effect op flak/rocket/elk ander wapen, elke map). Distances zijn
wapen-fysica-constanten (bal-snelheid/beam-range), niet map-specifiek → from-scratch leerbaar.

**Implementatie (nieuwe reward-component, volledig gewired):**
- `ShockComboAimReward` (dense, owner SHOOTING, routeert FIRE-channel via SHOT_EVENT key zoals
  combo_event). Reward = `w · normAlign · enemyFactor` waar normAlign = aim-cosine-naar-bal boven
  drempel, enemyFactor = exp(-balAfstandTotEnemy/sigma). Gated: fireActive, bal op mid-range
  (geen point-blank self-splash), enemy binnen context-range (anti-farm: geen combo'en op muur).
- Hergebruikt aim-conventie via nieuwe `FireModeAimTargeting.aimCosineToPoint(frame, point)`.
- `shock_combo_event` event_bonus 0.5 → 2.0 (geslaagde combo duidelijker bekrachtigd).

**Config (rewards.json default — generaliseert overal):**
- weight=0.12, min_aim_cos=0.95, min_ball_dist=120, max_ball_dist=1600,
  enemy_context_range=2000, ball_enemy_sigma=400.

**Deploy-strategie:** game-servers ONLY (hosts-filter excl. 4090) → trainer + 200k buffer blijven
intact; alleen experience-generatie/rewards veranderen. Buffer ververst zichzelf in ~10-20 min.

**Meetplan:** track `shockComboEvent=` (geslaagde combos, ÷2.0 nu) + nieuwe `shockComboAim=`
(combo-aim-shaping activiteit) per window over tijd, plus PLAYER_SCORES frags/accuracy. North-star =

## Round 4 (vervolgdag, user wakker) — proxy-farm-diagnose + aim-reward herontwerp

**Diagnose (4070, hele-nacht-log 152k windows chronologisch gebucket via `combo_trend_probe.py`):**

  zónder de combo te voltooien.

**Herinterpretatie van de Segment 1-5 "self-play-equilibrium maskeert skill"-hypothese:** een puur
dodge-equilibrium houdt de conversie *stabiel*; een **dalende** conversie bij stijgende aim+alt is de
handtekening van farming (proxy losgekoppeld van doel). De nacht-return-winst (+8.7%) zat dus deels in
geoogste aim-reward + echte shotOnTarget-winst (+19%), niet in betere combo-conversie. Absolute
combo-skill moet alsnog via een vaste benchmark vs stock bots (aanbeveling #1) bevestigd worden.

**Root cause:** (1) aim-reward betaalt voor RICHTEN zonder VOLTOOIEN; (2) `ball_enemy_sigma_uu`=300 gaf
krediet tot ~900 UU terwijl `combo_radius`=250 → beloning voor ballen te ver voor damage; (3)
reward-massa **13:1** (aim 3.43 vs combo_event 0.26/window) → gradient wijst naar richten i.p.v. voltooien.

- `ball_enemy_sigma_uu` **300 → 120** (gekoppeld aan `combo_radius`=250: enemyFactor @250UU=0.12,
  @120UU=0.37, @60UU=0.61 → krediet concentreert bínnen echte combo-bereik).
- Net-effect: geschatte aim/window valt van 3.43 → ~0.2-0.3 bij ongewijzigd gedrag → balans t.o.v.
  `shock_combo_event` (2.0) kantelt van 13:1 naar ~1:1 → voltooiing wordt de dominante prikkel; de
  bot moet ballen ÉCHT op de enemy plaatsen om aim-reward te behouden = zelfde richting als de combo.
