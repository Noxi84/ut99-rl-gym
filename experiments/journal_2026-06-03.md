## Round 3 — beam-detonatie-gate (geïmplementeerd, wacht op deploy)

**Wijziging (alleen DETECTIE, geen weights — interpreteerbare meting):**
- Nieuwe helper `FireModeAimTargeting.beamMissDistanceUu(frame, worldPoint)`: loodrechte afstand
  punt↔kijk-straal (afstandsonafhankelijk, i.t.t. een cosine-drempel).
- `ShockComboEventReward`: per vanished bal nu een gate vóór de enemy-scan — telt alleen als
  (a) primary `fireActive` op prev|curr ÉN (b) `beamMissDistanceUu(prev, ball) <= beam_hit_tolerance_uu`.
- `rewards.json` → `shock_combo_event.beam_hit_tolerance_uu = 45.0` (≈ ShockProj-collision +
  jitter-marge; ruim gestart om false-negatives/geen-leersignaal te vermijden).
- `ShockComboEventParams` + `ShockComboEventModule` (parse) uitgebreid. Compileert (mvn -pl java-rewards -am).

**Hypothese:** met de schone event-gate wijzen sparse event (10) + dense aim (0.2) beide naar de
échte combo. Alt-spam levert geen event meer → de prikkel om te spammen verdwijnt.

**Meet de volgende nacht (combo_trend_probe.py op 4070):**
- combo-presence zakt EERST naar de échte baseline (verwacht; 18% was opgeblazen).
- conversie combo/alt moet voor het eerst STOPPEN met dalen / STIJGEN. ← primaire succesmaat.
- alt/win moet stoppen met klimmen.

**Vervolg-tuning (één variabele per ronde):**
- conversie stijgt maar presence ~0 / alt stort in → gate te streng (frame-jitter) → verhoog
  beam_hit_tolerance_uu (45→60) of voeg curr-frame bal-fallback toe.
- conversie stijgt, presence redelijk, maar discovery traag → verhoog `shock_combo_aim.weight` (0.2→0.35).
- alt-spam blijft lonen → verlaag beam_hit_tolerance_uu (45→30).

de game-servers, bots herstarten; trainer pikt via replay-buffer op.

# CURRICULUM GEBOUWD (2026-06-04 ochtend, user koos optie C)

**Nieuwe dense reward `shock_combo_curriculum_shaping`** (PBRS): Φ(s)=max over eigen live ShockProj van
exp(−beamMiss/beam_sigma)·exp(−ballEnemyDist/enemy_sigma); reward=weight·clip(Φ(curr)−Φ(prev)) met
continuïteits-guard. Beloont de bot voor het GELEIDELIJK dichter op een eigen bal beamen die een vijand
nadert — de ontbrekende fijne precisie/timing. Files: `combat/shockcombocurriculum/` (Params/Reward/
Module) + RewardId + RewardSignal + RewardCatalog-accessor + JointRewardDecompositionStrategy (FIRE-head)
+ **RewardComputer (expliciete aanroep)** + rewards.json. Params: weight 2.0, beam_sigma 120, enemy_sigma
150, max_ball_enemy_range 800, clip 0.1.

**Debug-jacht (4 verkeerde sporen) — curriculum gaf eerst 0.000:**
1. beam_sigma 40 te streng → Φ verwaarloosbaar (40→120). Niet de hoofdoorzaak.

**Root cause (UT99-bron, `Botpack/ShockProj.uc`):** de combo is `SuperExplosion()` =

aparte combo-damage-type (de oude `DamTypeShockCombo`-aanname klopte niet). De oude detectie (bal-tracking
